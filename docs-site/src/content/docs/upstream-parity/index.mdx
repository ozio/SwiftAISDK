# Upstream Parity

SwiftAISDK tracks the provider-facing packages in Vercel AI SDK. The working documents in the repository are intentionally operational:

- `Docs/CoreV6Parity.md` tracks core and error-surface parity.
- `Docs/ProviderVersionLedger.md` records upstream package baselines.
- `Docs/ProviderCapabilityMatrix.md` records provider capabilities.
- `Docs/UpstreamSync.md` explains how to compare upstream packages and port behavior.

Use this section when you need to understand whether a behavior is intentionally Swift-native, still missing, or out of scope.

## Product Rule

Parity does not mean copying every TypeScript name. SwiftAISDK keeps upstream concepts recognizable while preserving Swift conventions, typed errors, async sequences, and Swift-native UI/session shapes.
